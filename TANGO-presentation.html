  4. ASCE — Adaptive Signal Control Engine

    Action-gate residual MAPPO with centralized training

    Person-Time-Loss (KPI)

    Σdelayi×occi — bus = 30, car = 1.3 persons

    Reward (person_objective)

    -log(1+Σwti·occiΣocci)+0.25·Jns,ew

    Fairness

    Jns,ew = Jain index over N/S vs E/W person-delay. Balances approach equity.

  6. Key Technical Decisions

    Three decisions that closed the gap

    Action-Gate Residual Warm-start from MP, learn when to deviate. Gate: 2% → 30%.
    Person-Weighted Reward Per-person delay norm. Removed throughput term that dominated gradients.
    4-Scenario Curriculum Round-robin AM/PM/surge/midday. 8 workers, LR × 1/√8.
  7. Reference Design (Not in Final Control Loop)

    PIRA surrogate for fast corridor what-if planning

    • Fast disruption what-if layer without full SUMO reruns.
    • Input is scenario context and per-intersection traffic state.
    • Output is corridor impact estimate and signal timing recommendations.
    none construction lane_closure transit_line major_event
    Graph Input (scenario, time_step) graph snapshots with intersections as nodes
    Backbone Node MLP (128), 3 GATConv layers with 4 heads, residual LayerNorm
    Dual Heads Impact head for delay, throughput, queue_total. Timing head for green_ns_s and green_ew_s.
    Required Inputs
    • Requires osm.net.xml, dataset.parquet, and a trained checkpoint.
    • intersection_map.csv is optional.
    Features and Topology
    • 17 node features: 10 traffic, 7 scenario.
    • Capacity set to 0 removes affected edges.
    Training Protocol
    • MSE with Adam (1e-3), gradient clip 1.0.
    • Default curriculum, early stop patience 15, split by scenario_id.
    Proposal Targets
    • MAPE at most 10 percent and R^2 at least 0.85.
    • Inference under 5 s per scenario.
