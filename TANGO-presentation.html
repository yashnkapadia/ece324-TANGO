  3. Data Flow

    From Raw Feeds to Timing Actions

    TANGO uses a strict signal operations pipeline so every recommendation can be traced from source telemetry through model assumptions to deployment outcomes.

    Step 1

    Ingest

    TMC traces, detector counts, and scenario metadata are synchronized in time.

    Step 2

    Engineer

    Queue proxies, phase pressure, and turn penalties become model features.

    Step 3

    Train

    Scenario-aware predictors estimate delay, throughput, and progression quality.

    Step 4

    Simulate

    SUMO runs stress recommendations under baseline, peak, and disrupted demand.

    Step 5

    Explain

    Decision cards summarize expected gains, confidence, and operational caveats.

    Step 6

    Iterate

    Feedback loops retune parameters as corridor signatures drift by daypart.

  4. System Design

    Modular and Reproducible by Construction

    Training, simulation, and reporting are separated into explicit contracts. This keeps experiments fast while preserving reproducibility for final results.

    asce/trainers

    Candidate model generation with confidence-aware output bundles.

    modeling

    Scenario evaluators for delay, queue persistence, and cycle stability.

    sumo/scenarios

    Network demand variants for baseline, incidents, and peak-hour shifts.

    reports

    Operator-facing storylines that translate model choices into actions.

    Engineering Principles
    Data Contracts

    Every layer has fixed schema boundaries to prevent hidden coupling.

    Scenario Registry

    Comparisons remain fair because interventions run on shared scenario definitions.

    Metrics First

    Delay, progression hit-rate, and queue tails drive model acceptance criteria.

    Script + Notebook Parity

    Exploratory and reproducible workflows stay aligned for final delivery.
