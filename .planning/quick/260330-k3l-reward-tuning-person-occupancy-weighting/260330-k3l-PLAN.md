# Quick Task: Reward Tuning

## Task 1: Update reward function in traffic_metrics.py

**Files:** `ece324_tango/asce/traffic_metrics.py`

**Changes:**
1. Import `occupancy_for_vehicle_type` from `kpi.py`
2. Add `_person_weighted_edge_metrics()` — queries per-vehicle waiting time + type on approach edges, weights by occupancy (30 for bus/tram, 1.3 for car)
3. Add `person_delay`, `person_throughput`, `person_delay_ns`, `person_delay_ew` fields to `IntersectionMetrics`
4. Compute person-weighted fields in `compute_metrics_for_agent()` using per-vehicle TraCI queries
5. Update `rewards_from_metrics()` objective mode:
   - Delay term: `log(1 + person_delay / max(1, person_throughput))` (per-person normalized)
   - Throughput term: `log(1 + person_throughput)` (person-weighted)
   - Fairness term: local Jain index over `[person_delay_ns, person_delay_ew]` (per-intersection approach balance)
   - Handles zero-delay case (both approaches zero → fairness = 1.0)
