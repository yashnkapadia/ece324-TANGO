## Directory Purposes

**ece324_tango/:**
- Purpose: Python package containing ASCE pipeline (training, evaluation, baselines, analysis)
- Contains: 23 Python modules across 3 sub-packages
- Key files: `config.py` (paths), `__init__.py` (package exports)

**ece324_tango/asce/:**
- Purpose: Core ASCE functionality—environment adapters, control policies, metrics
- Contains: env.py (SUMO wrapper), schema.py (data contracts), mappo.py (policy), baselines.py (heuristics), traffic_metrics.py/kpi.py (rewards), runtime.py (utilities), obs_norm.py (normalization)
- Key files: `env.py`, `mappo.py`, `traffic_metrics.py`

**ece324_tango/asce/trainers/:**
- Purpose: Training/evaluation backend implementations
- Contains: `base.py` (abstract backend + dataclasses), `local_mappo_backend.py` (full implementation), `factory.py` (backend selector), `noise_control.py` (output suppression)
- Key files: `local_mappo_backend.py` (orchestrates training loop)

**ece324_tango/modeling/:**
- Purpose: CLI entry points for training and evaluation
- Contains: `train.py` (training CLI), `predict.py` (evaluation CLI)
- Key files: Both expose `main()` via typer app

**sumo/network/:**
- Purpose: SUMO network and topology files
- Contains: `osm.net.xml` (Toronto OSM corridor network, 12 signalized intersections), `osm.net.xml.gz` (compressed), additional GIS overlays (poly, bbox)
- Generated by: SUMO OSM importer; not committed source

**sumo/demand/:**
- Purpose: Vehicle flow patterns for simulation
- Contains: `demand.rou.xml` (TMC-calibrated, 70 car flows 0-3600s), `random_trips.rou.xml` (synthetic)
- Format: SUMO route XML with <flow> elements

**sumo/config/:**
- Purpose: SUMO simulation configuration
- Contains: `baseline.sumocfg` (simulation parameters: network, demand, timestep, duration)

**data/processed/:**
- Purpose: Rollout samples and engineered features from training/evaluation
- Contains: `asce_rollout_*.csv` (trajectory data with ASCE_DATASET_COLUMNS), `tmc_parsed.csv` (TMC demand data)
- Key files: `asce_rollout_toronto_demand.csv` (full training rollout), `asce_rollout_toronto_demand_time_loss.csv` (alternative mode)

**models/:**
- Purpose: Trained policy checkpoints
- Contains: `asce_mappo_toronto_demand.pt` (PyTorch .pt file with actor, critic, normalizer state)
- Format: PyTorch dict with keys for actor.state_dict(), critic.state_dict(), normalizers

**reports/results/:**
- Purpose: Evaluation metrics and error logs
- Contains: `asce_train_episode_metrics_*.csv` (episode-level: mean_global_reward), `asce_eval_metrics_*.csv` (seed-level: KPIs per controller), `error_events.jsonl` (non-fatal exceptions)
- Format: CSV for metrics, JSONL for errors (one JSON object per line)

**reports/figures/:**
- Purpose: Generated interim report figures
- Contains: `interim_objective_results.png` (2x2 subplot: training curve, KPI summary plots)
- Format: PNG, 11x8.5 inches, reproducible via pixi task

**tests/:**
- Purpose: Unit and integration tests (19 files covering core modules)
- Contains: 19 .py test files + conftest.py
- Key coverage: mappo_core, baselines, runtime, obs_norm, traffic_metrics, kpi, backend evaluation guards

## Naming Conventions

**Files:**
- `*.py`: Python source modules (no underscores between words unless part of name)
- `train.py`, `predict.py`: Entry point modules (action-oriented names)
- `mappo.py`, `baselines.py`: Functional modules (policy/baseline names)
- `test_*.py`: Test files (test_ prefix + module under test)
- `*.rou.xml`: SUMO route files (demand patterns)
- `*.net.xml`: SUMO network files (topology)
- `*.sumocfg`: SUMO configuration files

**Directories:**
- `asce/`: Acronym-based (Adaptive Signal Control Engine)
- `trainers/`: Plural for directory of implementations
- `data/`, `models/`, `reports/`: Artifact categories
- `network/`, `demand/`, `config/`: SUMO sub-categories

**Python Classes:**
- Actor, Critic, MAPPOTrainer: PascalCase, descriptive (what they are)
- FixedTimeController, MaxPressureController: PascalCase, strategy names
- ObsRunningNorm: PascalCase, algorithm-focused
- TrainConfig, EvalConfig: PascalCase, dataclass configs
- LocalMappoBackend: PascalCase, backend name + location
- KPITracker: PascalCase, tool name

**Python Functions:**
- `create_parallel_env()`: snake_case, verb-first
- `flatten_obs_by_agent()`: snake_case, descriptive
- `compute_metrics_for_agents()`: snake_case, verb-first
- `get_backend()`: snake_case, factory pattern

**Python Variables:**
- `obs`, `gobs`: Shortened (observation, global observation)
- `cfg`: Shortened (config)
- `env`: Shortened (environment)
- `action_dims`, `n_actions`: Descriptive plurals/counts
- `ordered_agents`: Explicit (agents in specific order)

**CSV Columns:**
- `intersection_id`, `time_step`: snake_case
- `queue_ns`, `queue_ew`: Direction abbrevs (NS=north-south, EW=east-west)
- `avg_speed_ns`, `arrivals_ew`: Metric + direction
- `person_time_loss_s`: Units in name (seconds)

## Where to Add New Code

**New Control Policy:**
- Primary code: `ece324_tango/asce/` → new file `policy_name.py`
- Implementation: Inherit from `AsceTrainerBackend` in `trainers/base.py`
- Registry: Add to `get_backend()` factory in `trainers/factory.py`
- Tests: `tests/test_policy_name.py`
- CLI: Add new Typer app in `modeling/` or extend `train.py`

**New Baseline Controller:**
- Primary code: `ece324_tango/asce/baselines.py` → add new @dataclass
- Method signature: `actions(observations: Dict[str, np.ndarray]) -> Dict[str, int]` + optional `env` kwarg
- Tests: `tests/test_baseline_*.py`
- Integration: Update `LocalMappoBackend.evaluate()` to include new controller

**New Metric or Reward Mode:**
- Metrics: `ece324_tango/asce/traffic_metrics.py` → add function or extend `RewardWeights`
- KPIs: `ece324_tango/asce/kpi.py` → extend `EpisodeKPI` dataclass if needed
- Reward mode: Add to `_VALID_REWARD_MODES` in `train.py` and `predict.py`, implement in `LocalMappoBackend.train()`
- Tests: `tests/test_traffic_metrics.py` or new test file

**New Utility/Helper:**
- Shared helpers: `ece324_tango/asce/runtime.py` (for runtime logic) or new module in `asce/`
- Normalization: `ece324_tango/asce/obs_norm.py` (observation processing)
- Tests: `tests/test_utility_name.py`

**New Analysis/Visualization:**
- Standalone script: `ece324_tango/feature_name.py` with Typer app + `pixi` task
- Plotting: `ece324_tango/plots.py` → add new figure function with render_figure pattern
- No tests required (exploratory code)

**Environment Modifications:**
- SUMO network changes: Edit `sumo/network/osm.net.xml` (or regenerate from OSM)
- Demand changes: Add/edit `sumo/demand/*.rou.xml`
- Config changes: Update `sumo/config/baseline.sumocfg` or create scenario-specific config
- Wrapper logic: `ece324_tango/asce/env.py` → modify `create_parallel_env()` or helper functions

## Special Directories

**logs/xuance_mappo/:**
- Purpose: Training logs from xuance MAPPO comparison (older experiment)
- Generated: During training with verbose logging
- Committed: No (in .gitignore)

**notebooks/:**
- Purpose: Jupyter notebooks for exploratory analysis
- Generated: User-created during development
- Committed: Typically no (binary format, state inconsistent)

**.pixi/envs/default/:**
- Purpose: Pixi-managed Python environment
- Generated: By pixi install
- Committed: No (in .gitignore)

**reports/results/error_events.jsonl:**
- Purpose: Non-fatal exception log
- Generated: During training/eval runs via `report_exception()`
- Committed: No (grows unbounded, specific to environment)

**models/:**
- Purpose: Trained checkpoints
- Generated: By `train.py`
- Committed: Selectively (large files, binary)
